# UI / UX Guideline

## When to use

Use this guideline when:

- creating a new tool page
- modifying layout or UI structure
- adjusting spacing / colors / typography
- designing result display

Do NOT use this for:

- business logic
- calculation formulas
- backend design

## 設計定位

本網站為「實用型小工具集合」，核心原則：

- 極簡（Minimal）
- 工具導向（Utility-first）
- 結果優先（Result-first）
- 手機優先（Mobile-first）
- 快速理解（Zero learning cost）

不走：

- 不做內容型網站
- 不堆疊過多說明文字
- 不做複雜互動

---

## 整體視覺風格

### Layout

- Max width：960px
- 置中排列
- 上下留白清楚（但不鬆散）

---

### 卡片設計（核心元件）

- 白底
- 淡邊框（#e5e7eb）
- 圓角（12px~16px）
- 適度 padding（16~24px）
- 區塊間距一致（16~24px）

---

### 字級層次

- Title：20~24px（工具名稱）
- Section：16~18px（區塊標題）
- Body：14~16px
- 說明文字：13~14px（偏淡）

---

### 顏色策略

- 主色：中性（黑 / 灰）
- 強調色：僅用於「結果 / 狀態」

狀態顏色：

- 綠：成功 / 安全 / 可行
- 黃：中立 / 需注意
- 紅：風險 / 不建議 / 失敗

---

## 互動原則

### Input

- 欄位數量最少化
- 預設值要合理
- 支援「範例填入」

---

### Button

- 主按鈕：明顯（背景色）
- 未選狀態要可辨識（避免太接近）
- Hover / active 要有回饋

---

### Result（最重要）

結果區必須：

- 視覺最強
- 一眼看懂
- 不需閱讀說明

內容包含：

- 結論（大字）
- 關鍵數值
- 狀態顏色

---

### 狀態提示

避免只用文字：

- 搭配顏色
- 或 icon / 標籤
- 或區塊背景

---

## 說明區設計

原則：

- 短
- 可略讀
- 不影響主流程

避免：

- 大段文字
- 教學文章風格

建議：

- 條列式
- 關鍵句
- FAQ 型

---

## Footer

- 簡潔
- 不干擾主內容
- 固定結構：

內容：

- IG
- 貼圖
- 聯絡
- 版權

---

## 響應式（RWD）

- 手機優先設計
- 單欄
- 按鈕可點擊區域大
- 不需要橫向 scroll

---

## 設計一致性原則（非常重要）

所有工具必須：

- 使用相同卡片樣式
- 使用相同 spacing 規則
- 使用相同按鈕風格
- 使用相同結果呈現方式

目標：

👉 使用者一眼就知道「這是同一個網站的工具」

---

## 設計優先順序

1. 是否一眼看懂結果
2. 是否操作簡單
3. 是否手機好用
4. 才是美觀
